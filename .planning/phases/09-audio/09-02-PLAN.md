  <name>Task 1: Wire sound effects to game events</name>
  <files>src/systems/audio.js</files>
  <action>
Inside `createAudioSystem()`, after the audioCtx resume logic, add event listeners that trigger sounds on game events:

1. **AUD-01 — Pop on correct key press:**
```js
events.on('OBSTACLE_DESTROYED', () => { playPop(); });
```

2. **AUD-02 — Thud on obstacle miss:**
```js
events.on('LIFE_LOST', () => { playThud(); });
```
Use LIFE_LOST (not OBSTACLE_MISSED) because LIFE_LOST is the semantically correct "damage happened" signal — same event VFX uses for screen flash. This ensures audio and visual feedback are in sync.

3. **AUD-03 — Game over descending tone:**
```js
events.on('STATE_CHANGE', ({ state }) => {
  if (state === 'game_over') {
    playGameOver();
    // Music stop is already handled in Plan 01's STATE_CHANGE listener
  }
});
```
Note: If Plan 01 already has a STATE_CHANGE listener for music stop, consolidate into a single listener that handles both game-over sound AND music stop. Do NOT register duplicate STATE_CHANGE listeners.

4. **AUD-06 — Level-up jingle:**
```js
events.on('LEVEL_UP', () => { playLevelUp(); });
```

Important: All these event subscriptions go inside `createAudioSystem()` so they're registered at init time. The sound functions themselves already guard with `if (!audioReady) return`, so events firing before first interaction are silently ignored.

  <action>
1. In src/systems/audio.js, inside `createAudioSystem()`, add a keydown listener for the M key to toggle music:

```js
window.addEventListener('keydown', (e) => {
  if (e.key === 'm' || e.key === 'M') {
    toggleMusic();
  }
});
```

This listener is separate from the audio resume listener. It only calls toggleMusic() which internally handles enabling/disabling the music oscillator and updating the musicGain.

2. In src/main.js render function, add a small HUD hint for the music toggle. In the PLAYING state render section (after renderHUD), add a subtle text hint:

After `renderHUD(ctx, w);` add:
```js
// Music toggle hint (bottom-right, dim)
ctx.save();
ctx.font = '10px monospace';
ctx.fillStyle = COLORS.PALETTE.DIM;
ctx.textAlign = 'right';
ctx.fillText('[M] Music', w - 8, getHeight() - 8);
ctx.restore();
```

This is a minimal, unobtrusive hint that doesn't require importing audio module into main.js for state checking. It's always visible in PLAYING state.
