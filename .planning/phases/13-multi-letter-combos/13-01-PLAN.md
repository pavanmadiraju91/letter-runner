<task type="auto">
  <name>Task 2: Update spawner for level-gated combo spawning with uniqueness</name>
  <files>src/systems/spawner.js</files>
  <action>
  Refactor updateSpawner to support combo obstacle spawning:

  1. Import `COMBO` from config and `getLevel` from difficulty.js.

  2. After the existing letter uniqueness logic, add combo decision logic:
     - Collect ALL used letters across all active obstacles: for single obstacles use `obs.letter`, for combo obstacles use all entries in `obs.letters`.
     - Determine if this spawn should be a combo:
       a. Count active combos: `active.filter(o => o.isCombo).length`
       b. If active combos >= COMBO.MAX_ON_SCREEN (1), force single-letter.
       c. If level >= COMBO.MIN_LEVEL_3LETTER, roll SPAWN_CHANCE_3LETTER first; if hit, spawn 3-letter combo.
       d. Else if level >= COMBO.MIN_LEVEL_2LETTER, roll SPAWN_CHANCE_2LETTER; if hit, spawn 2-letter combo.
       e. Otherwise, spawn single-letter as before.
     - For combo spawns: pick N unique letters (not in usedLetters set), set `obs.isCombo = true`, `obs.letters = [picked]`, `obs.progress = 0`, `obs.letter = ''`, `obs.width = COMBO.WIDTH_PER_LETTER * N`.
     - For single spawns: keep existing behavior, ensure `obs.isCombo = false`, `obs.letters = []`, `obs.progress = 0`.

  3. Update the usedLetters Set construction at the top of the spawn logic to iterate all active obstacles and collect:
     - `obs.letter` if it's a single (non-empty)
     - All entries in `obs.letters` if it's a combo

  4. When picking N letters for a combo, pick them one at a time, adding each to usedLetters before picking the next, to guarantee internal uniqueness within the combo itself.

  5. If there aren't enough available letters for the combo size (e.g., need 3 but only 2 available), fall back to single-letter spawn.

  IMPORTANT: Do NOT change the obstacle release/cleanup logic or the offscreen detection. Combo obstacles use the same pool and same movement. The width being larger means they leave the screen later (x + width < 0), which is already handled by cleanupOffscreen using obs.width.

  Update the OBSTACLE_MISSED event in cleanupOffscreen (obstacle.js) to emit `letters: obs.letters` for combo obstacles so downstream systems know what was missed.
  </action>

  Run: `grep -n "OBSTACLE_MISSED" src/entities/obstacle.js` — should show letters field in emit.
  </verify>
