  <action>
  Rewrite the KEY_PRESS handler inside initMatcher to handle both single and combo obstacles:

  The matching algorithm:
  1. Get all active obstacles in the danger zone (same zone logic: x >= dangerLeft && x <= dangerRight).
  2. Sort/iterate from rightmost (closest to player) first — same priority as before.
  3. For each obstacle in the danger zone:
     a. **Single-letter obstacle** (obs.isCombo === false):
        - If obs.letter === key: emit OBSTACLE_DESTROYED with `{ letter: obs.letter, x: obs.x, y: obs.y, isCombo: false, comboSize: 1 }`, release obstacle, return.
     b. **Combo obstacle** (obs.isCombo === true):
        - If obs.letters[obs.progress] === key:
          - Increment obs.progress.
          - If obs.progress === obs.letters.length: combo complete! Emit OBSTACLE_DESTROYED with `{ letter: obs.letters.join(''), x: obs.x, y: obs.y, isCombo: true, comboSize: obs.letters.length }`, release obstacle, return.
          - Otherwise: partial match, the combo advances. Return immediately (key consumed).
        - Else (wrong key mid-combo, since the combo is in the zone and was the target):
          - Reset obs.progress = 0.
          - Emit WRONG_KEY with `{ key, comboReset: true }`.
          - Return immediately (key consumed).

  4. If no obstacle in zone was matched or targeted:
     - If there were obstacles in the zone (hasObstaclesInZone): emit WRONG_KEY with `{ key, comboReset: false }`.

  IMPORTANT PRIORITY RULE: Combo obstacles take priority over single-letter obstacles when both are in the danger zone. Rationale: the combo is time-sensitive (multiple keys needed), and there can only be 1 combo on screen. Check combo obstacles first, then single-letter.

  Implementation approach:
  - First pass: find any combo obstacle in the danger zone. If found, that's the target.
  - If the combo's next letter matches the key: advance or destroy.
  - If the combo's next letter does NOT match: check if any single-letter obstacle matches the key. If yes, destroy the single. If no, it's a wrong key against the combo — reset progress.
  - Second pass (no combo in zone): standard single-letter matching as before.

  This ensures typing a single-letter that's on screen doesn't incorrectly reset a combo, while wrong keys that don't match anything do reset the combo.
  </action>
