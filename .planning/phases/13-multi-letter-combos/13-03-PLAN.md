<task type="auto">
  <name>Task 2: Update scoring to apply combo multipliers</name>
  <files>src/systems/score.js</files>
  <action>
  Import `COMBO` from '../config.js' at the top of score.js.

  Modify the OBSTACLE_DESTROYED event handler in createScore():

  Current code:
  ```js
  events.on('OBSTACLE_DESTROYED', () => {
    score += SCORE.DESTROY_POINTS * currentLevel;
  });
  ```

  Updated code:
  ```js
  events.on('OBSTACLE_DESTROYED', (payload) => {
    let comboMultiplier = 1;
    if (payload && payload.isCombo && payload.comboSize) {
      if (payload.comboSize === 3) {
        comboMultiplier = COMBO.MULTIPLIER_3LETTER;
      } else if (payload.comboSize === 2) {
        comboMultiplier = COMBO.MULTIPLIER_2LETTER;
      }
    }
    score += SCORE.DESTROY_POINTS * currentLevel * comboMultiplier;
  });
  ```

  This applies the 2.5x or 4x multiplier ON TOP of the existing level multiplier. A 3-letter combo at level 5 gives: 10 * 5 * 4 = 200 points (vs 50 for a single letter).

  The payload defensively checks for existence so old-style events (without isCombo) still work — backward compatible.
  </action>
  <verify>
  Run: `node -c src/systems/score.js` — no syntax errors.
  Run: `grep -n "comboMultiplier\|COMBO\|comboSize" src/systems/score.js` — shows multiplier logic.
  </verify>
  <done>Scoring awards 2.5x for 2-letter and 4x for 3-letter combo completions, stacking with level multiplier.</done>
