Create src/screens/game-over.js:

Module-level state:
- let phase = 'display' (phases: 'display' | 'name-entry' | 'done')
- let initials = '' (0-3 characters)
- let finalScore = 0
- let personalBest = 0
- let qualifies = false
- let flashTimer = 0 (for cursor blink in name entry)

createGameOverScreen():
- Subscribe to events (via events import):
  - STATE_CHANGE: when state becomes 'game_over':
    - Set finalScore from getScore()
    - Set personalBest from getPersonalBest()
    - Set qualifies from qualifiesForLeaderboard(finalScore)
    - Set phase to qualifies ? 'name-entry' : 'done'
    - Reset initials = ''
  - KEY_PRESS with payload: during game_over state and phase === 'name-entry':
    - Capture key from payload.key (the raw key string)
    - If key is a single letter (A-Z, a-z): append uppercase to initials (max 3)
    - If key is 'Backspace': remove last char from initials
    - If key is 'Enter' and initials.length === 3: call insertScore(initials, finalScore), set phase = 'done'
    - IMPORTANT: When phase is 'name-entry', do NOT allow game restart (block the default KEY_PRESS restart handler)

updateGameOverScreen(dt):
- flashTimer += dt (used for cursor blink)

renderGameOverScreen(ctx, width, height):
- Fill background: rgba(0,0,0,0.85) over full canvas
- Draw "GAME OVER" title: bold 36px monospace, white, centered at y=height*0.15
- Draw final score: "SCORE: {finalScore}", bold 28px monospace, white, at y=height*0.25
- Draw personal best line:
  - "BEST: {personalBest}", 18px monospace, #ffcc00, at y=height*0.32
  - Delta: if finalScore > personalBest: "+{delta} NEW BEST!" in #00ff88
  - Else: "-{delta}" in #ff4466
- Draw leaderboard (getLeaderboard()):
  - Title "TOP 10" at y=height*0.40, 16px monospace, #aaaaaa
  - Loop top 10: render "NN. AAA  SCORE" rows
    - 14px monospace, y starting at height*0.45, each row +20px
    - Highlight the player's new entry (if inserted) in #00ffcc
    - Others in #cccccc
    - Rank number left-aligned, initials centered, score right-aligned (use fixed-width columns)
- If phase === 'name-entry':
  - Draw "ENTER YOUR INITIALS" at y=height*0.88, 18px monospace, #ffffff
  - Draw initials with underscores for remaining: e.g. "A B _" with cursor blink on current position
    - Use Math.floor(flashTimer * 2) % 2 for blink
  - Draw "(ENTER to confirm)" hint below, 12px monospace, #888888
- If phase === 'done':
  - Draw "Press any key to play again" at y=height*0.90, 18px monospace, white, with blink

Modify src/main.js:
- Import { createGameOverScreen, updateGameOverScreen, renderGameOverScreen } from './screens/game-over.js'
- Call createGameOverScreen() during initialization
- Remove the existing inline game-over overlay rendering (the ctx.fillText('GAME OVER'...) block in render())
- In render(): when state is GAME_OVER, call renderGameOverScreen(ctx, w, h) instead
- In update(): when state is GAME_OVER, call updateGameOverScreen(dt)
- Modify KEY_PRESS handler for game over state:
  - The game-over screen's createGameOverScreen already handles key capture during name-entry
  - Only call restartGame() if getGameOverPhase() returns 'done' (need to export this or use an event pattern)
  - Simplest approach: export a function isReadyToRestart() from game-over.js that returns phase === 'done'
  - In main.js KEY_PRESS handler: if GAME_OVER state AND isReadyToRestart(), then restartGame()

KEY DESIGN NOTE: The KEY_PRESS event during name-entry must NOT trigger restartGame(). The game-over screen captures alpha keys for initials. Only after name entry is complete (phase='done') should key presses restart the game.
