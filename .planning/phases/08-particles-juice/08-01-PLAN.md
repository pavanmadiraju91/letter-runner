```js
export const PARTICLES = {
  MAX_ACTIVE: 30,        // VFX-09: hard cap
  POOL_SIZE: 40,         // pre-allocate slightly more than max
  MIN_FPS: 30,           // VFX-10: skip threshold
};
```

Create src/systems/particles.js:

- Uses createPool from '../systems/pool.js' with a particle factory
- Particle shape: `{ x, y, vx, vy, life, maxLife, color, size }`
- `createParticleSystem()` — creates the internal pool (PARTICLES.POOL_SIZE initial), module-level ref
- `canSpawnParticles()` — returns false if pool active count >= PARTICLES.MAX_ACTIVE OR if isFPSLow() is true. This is THE performance gate (VFX-09 + VFX-10).
- `spawnParticles(x, y, count, config)` — spawns up to `count` particles at (x,y). Config has: { color, minSpeed, maxSpeed, minSize, maxSize, minLife, maxLife }. For each particle: random angle (0..2PI), random speed between min/max, vx=cos*speed, vy=sin*speed, random size, random life. Clamp actual spawned count so active never exceeds MAX_ACTIVE.
- `updateParticles(dt)` — iterate active particles: x += vx*dt, y += vy*dt, life -= dt. If life <= 0, release back to pool. Add gravity: vy += 400*dt (downward pull for natural arc).
- `renderParticles(ctx)` — iterate active particles: alpha = life/maxLife, fillStyle = color, globalAlpha = alpha, fillRect(x, y, size, size). Use ctx.save/restore wrapping the entire loop (one save/restore, not per-particle).

Import isFPSLow from './fps-monitor.js' for the canSpawnParticles guard.
  </action>
  <verify>File exists, exports all 5 functions. Build succeeds: `npx vite build --mode production 2>&1 | grep -i error`. Verify MAX_ACTIVE=30 in config.</verify>
  <done>Particle system exists with 30-particle hard cap (VFX-09) and FPS < 30 skip guard (VFX-10). Pool pre-allocated. Gravity applied for natural arcs.</done>
