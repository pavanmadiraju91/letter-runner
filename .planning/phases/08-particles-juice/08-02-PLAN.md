  <action>
Create src/systems/vfx.js:

Module-level state (closure pattern consistent with lives.js):
- `playerFlashTimer = 0` — counts down from FLASH_DURATION
- `const FLASH_DURATION = 0.1` — 100ms white flash

Exports:

1. `createVFX()` — subscribes to events:
   - `OBSTACLE_DESTROYED`: calls spawnDestroyParticles(payload.x, payload.y) AND sets playerFlashTimer = FLASH_DURATION
   - `GAME_RESTART`: resets playerFlashTimer = 0

2. `updateVFX(dt)` — decrements playerFlashTimer by dt, clamps to 0.

3. `getPlayerFlash()` — returns playerFlashTimer > 0 (boolean). Used by render to apply white overlay.

Internal function `spawnDestroyParticles(x, y)`:
- Guard with canSpawnParticles() — skip entirely if false
- Call spawnParticles(x, y, 8, config) with:
  - color: randomly pick from [COLORS.PALETTE.MAGENTA, COLORS.PALETTE.CYAN, COLORS.PALETTE.YELLOW] per particle (pass a function or spawn 3 batches of ~3 particles each with different colors)
  - Actually simpler: call spawnParticles 3 times with count ~3 each, different colors. This keeps the API simple.
  - minSpeed: 80, maxSpeed: 200
  - minSize: 2, maxSize: 5
  - minLife: 0.3, maxLife: 0.8

Note: spawnParticles already handles the MAX_ACTIVE cap internally, so even if we request 9 total, it won't exceed 30.

Import: events, canSpawnParticles, spawnParticles, COLORS

In src/main.js:

1. Import: `import { createVFX, updateVFX, getPlayerFlash } from './systems/vfx.js';`
2. After createParticleSystem(), add: `createVFX();`
3. In update(), after `updateParticles(dt);`, add: `updateVFX(dt);`
4. In render(), modify the player rendering section. Currently it's just `renderPlayer(ctx, player);`. Change to:

```js
renderPlayer(ctx, player);
// VFX-05: white flash overlay on correct key press
if (getPlayerFlash()) {
  ctx.save();
  ctx.globalAlpha = 0.7;
  ctx.globalCompositeOperation = 'source-atop';
  // Actually source-atop won't work on immediate-mode canvas without a layer.
  // Simpler approach: draw a white rect over the player bounds
  ctx.fillStyle = '#ffffff';
  ctx.fillRect(player.x, player.y, player.width, player.height);
  ctx.restore();
}
```

Wait — even simpler and more visually correct for pixel art: draw a white-tinted version by overlaying a semi-transparent white rectangle exactly on the player bounds. This is crude but effective for a 32x32 sprite.

Actually the cleanest approach for a pixel sprite:
```js
renderPlayer(ctx, player);
if (getPlayerFlash()) {
  ctx.save();
  ctx.globalAlpha = 0.6;
  ctx.fillStyle = COLORS.PALETTE.WHITE;
  ctx.fillRect(player.x, player.y, player.width, player.height);
  ctx.restore();
}
```

This overlays white on the player area, making it flash. Simple, performant, and reads clearly as "player just did something good."
