<task type="auto">
  <name>Task 2: Create scrolling ground plane</name>
  <files>src/entities/ground.js</files>
  <action>
  Create `src/entities/ground.js` with three exports:

  1. `createGround()` — returns plain object:
     ```javascript
     { offset: 0, tileWidth: 64, y: 0, height: GAME.GROUND_HEIGHT }
     ```

  2. `updateGround(ground, dt, scrollSpeed)` — scrolls the ground:
     ```javascript
     ground.offset = (ground.offset + scrollSpeed * dt) % ground.tileWidth;
     ```
     Uses modulo wrap to keep offset bounded (no overflow over long play sessions).

  3. `renderGround(ctx, ground, canvasWidth, canvasHeight)` — renders the ground band:
     - Set `ground.y = canvasHeight - ground.height`
     - Draw filled rectangle across full width at ground.y in color `#1a1a2e`
     - Draw scroll indicators (dashes) in color `#2a2a4e`, spaced by `ground.tileWidth`, offset by `-ground.offset` to create scrolling motion:
       ```javascript
       const startX = -ground.offset;
       for (let x = startX; x < canvasWidth; x += ground.tileWidth) {
         ctx.fillRect(x, ground.y + 4, 32, 2);
       }
       ```

  Import GAME from config.js for GROUND_HEIGHT. The ground must visually scroll left to create the illusion of the player running right.
  </action>
  <verify>

  - `grep -q "export function renderGround" src/entities/ground.js` succeeds
  - `grep -q "tileWidth" src/entities/ground.js` succeeds (modulo wrap present)
  </verify>
  <done>Ground entity scrolls continuously with modulo wrap. Scroll indicators visually move left. Y is computed from canvas height making it responsive.</done>
