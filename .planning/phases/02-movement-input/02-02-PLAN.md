<task type="auto">
  <name>Task 1: Create generic object pool</name>
  <files>src/systems/pool.js</files>
  <action>
  Create `src/systems/pool.js` exporting `createPool(factory, initialSize = 20)`.

  The pool pre-allocates `initialSize` objects using the provided factory function. Returns an object with these methods:
  - `acquire()` — pops from available pool (or calls factory if pool empty), pushes to active array, returns the object
  - `release(obj)` — removes from active array via splice (max 4 active, so O(n) is trivial), pushes back to pool
  - `getActive()` — returns the active array reference (for iteration in update/render loops)
  - `releaseAll()` — moves all active objects back to pool (for game reset)
  - `stats()` — returns `{ active: N, available: M }` for debugging

  Key design choices:
  - Use `splice` for release (array is max 4 items, not worth a swap-remove)
  - Pre-allocate 20 to ensure NO runtime `new` during gameplay
  - `getActive()` returns the live array reference (avoid copy allocation)
  - No `new` keyword anywhere in the pool — factory returns plain objects
  </action>
  <verify>

  - `grep -q "acquire" src/systems/pool.js` succeeds
  - `grep -q "release" src/systems/pool.js` succeeds
  - `grep -q "getActive" src/systems/pool.js` succeeds
  </verify>
  <done>Generic pool pre-allocates objects and recycles them without runtime allocation. All five methods (acquire, release, getActive, releaseAll, stats) work correctly.</done>

<task type="auto">
  <name>Task 2: Create obstacle entity and spawner system</name>
  <files>src/entities/obstacle.js, src/systems/spawner.js</files>
  <action>
  **Part A — src/entities/obstacle.js:**

  Create with these exports:
  1. `createObstacleFactory()` — returns a factory function that creates plain obstacle objects:
     ```javascript
     { x: 0, y: 0, width: GAME.OBSTACLE_WIDTH, height: GAME.OBSTACLE_HEIGHT, letter: '', speed: 0, active: false }
     ```

  2. `updateObstacles(pool, dt)` — iterates `pool.getActive()`, moves each obstacle left: `obs.x -= obs.speed * dt`

  3. `cleanupOffscreen(pool)` — iterates active obstacles in reverse, releases any where `obs.x + obs.width < 0`. Emit `events.emit('OBSTACLE_MISSED', { letter: obs.letter })` before releasing (Phase 3 will subscribe to this for life loss). Iterate in reverse because releasing modifies the array.

  4. `renderObstacles(ctx, obstacles)` — for each obstacle:
     - Draw body: `ctx.fillRect(Math.round(obs.x), Math.round(obs.y), obs.width, obs.height)` in color `#ff2266`
     - Draw letter: bold 24px monospace, white (`#ffffff`), centered on obstacle using `textAlign: 'center'` and `textBaseline: 'middle'`. Use `Math.round()` on text position to prevent sub-pixel blur.

  Import GAME from config.js, events from core/events.js.

  **Part B — src/systems/spawner.js:**

  Create with these exports:
  1. `createSpawner(pool)` — returns spawner state: `{ pool, timer: 0 }`

  2. `updateSpawner(spawner, dt, scrollSpeed, groundY)` — timer-based spawning logic:
     - Increment `spawner.timer += dt`
     - If timer >= `GAME.SPAWN_INTERVAL`:
       - Reset timer to 0
       - Guard: if `pool.getActive().length >= GAME.MAX_OBSTACLES`, return
       - Guard: if last obstacle hasn't moved far enough from right edge (gap < `GAME.MIN_OBSTACLE_GAP`), return
       - Pick a random letter not already on screen (loop with max 26 attempts using Set of active letters)
       - Acquire from pool, set: `x = getWidth() + 10`, `y = groundY - GAME.OBSTACLE_HEIGHT`, `letter = pickedLetter`, `speed = scrollSpeed`, `active = true`

  Import GAME from config.js, getWidth from core/canvas.js.

  Letter uniqueness: build `Set` from `pool.getActive().map(o => o.letter)`, then loop random picks until an unused letter is found.
  </action>
