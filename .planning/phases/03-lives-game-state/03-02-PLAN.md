  <name>Task 1: Create state machine module</name>
  <files>src/core/state.js</files>
  <action>
Create src/core/state.js with a minimal state machine:

1. Export `STATES` object: `{ PLAYING: 'playing', GAME_OVER: 'game_over' }`.
   Note: No MENU state yet — that comes in Phase 6 (Screens & Flow). For now the game starts directly in PLAYING.

2. Internal `let currentState = STATES.PLAYING` variable.

3. `createStateMachine()` — subscribes to events:
   - `GAME_OVER`: sets currentState to STATES.GAME_OVER, emits `STATE_CHANGE` with `{ state: STATES.GAME_OVER }`.
   - `GAME_RESTART`: sets currentState to STATES.PLAYING, emits `STATE_CHANGE` with `{ state: STATES.PLAYING }`.

4. `getState()` — returns currentState.

5. `requestRestart()` — emits `GAME_RESTART` (called by input handler during GameOver).

The machine is intentionally simple. No transition guards beyond "if already in target state, ignore." Emitting STATE_CHANGE lets other systems react without coupling.
  </action>
  <verify>Read src/core/state.js — exports STATES, createStateMachine, getState, requestRestart. Subscribes to GAME_OVER and GAME_RESTART events.</verify>
  <done>State machine transitions PLAYING -> GAME_OVER on GAME_OVER event, and GAME_OVER -> PLAYING on GAME_RESTART event.</done>
</task>

<task type="auto">
  <name>Task 2: Wire state machine into main.js with restart logic</name>
  <files>src/main.js</files>
  <action>
Modify src/main.js to integrate the state machine and restart:

1. Import `createStateMachine`, `getState`, `STATES`, `requestRestart` from './core/state.js'.
2. Import `createLives`, `resetLives` from './systems/lives.js'.
3. Call `createStateMachine()` and `createLives()` during initialization.
4. Call `resetLives()` after createLives to set initial 3 lives.

5. Guard `update(dt)`:
   - At the top of update(), check `if (getState() !== STATES.PLAYING) return;`
   - This stops all movement during GameOver but keeps render() running (so game over text can display).

6. Add a `restartGame()` function that:
   - Calls `resetLives()` (back to 3).
   - Releases all active obstacles back to pool: `obstaclePool.getActive().slice().forEach(o => obstaclePool.release(o))`.
   - Resets the spawner timer (add a `resetSpawner(spawner)` call — will need a small addition to spawner.js or just reset `spawner.timer = 0`).
   - Calls `requestRestart()` to flip state back to PLAYING.

7. Listen for restart key during GameOver:
   - Subscribe to `KEY_PRESS` in main.js. In the handler: `if (getState() === STATES.GAME_OVER) { restartGame(); }`.

8. Render a simple "GAME OVER - Press any key" text when in GAME_OVER state:
   - In `render()`, after drawing the scene, if `getState() === STATES.GAME_OVER`:
     - Draw a semi-transparent dark overlay (`ctx.fillStyle = 'rgba(0,0,0,0.7)'; ctx.fillRect(0,0,w,h);`).
     - Draw "GAME OVER" centered in white, 36px bold.
     - Draw "Press any key to restart" below it in 18px.

Note: The restart is instant because we just reset counters and flip state — no reload, no re-init. This satisfies UI-13 (under 1 second).
  </action>
