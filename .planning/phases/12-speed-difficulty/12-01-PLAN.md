  <name>Task 1: Add SPEED config and rewrite difficulty.js for continuous acceleration</name>
  <files>src/config.js, src/systems/difficulty.js</files>
  <action>
In src/config.js, add a new SPEED section (keep DIFFICULTY for now — it still provides multiplier, tallObstacles, maxObstacles by level):

```js
export const SPEED = {
  BASE_SPEED: 140,          // px/s at run start — 240px danger zone in 1.7s
  MAX_SPEED: 280,           // px/s hard cap (2x base) — danger zone in 0.85s
  ACCELERATION: 2.0,        // px/s gained per second of play
  WARMUP_TIME: 2.0,         // seconds before first obstacle may spawn
  MIN_REACTION_MS: 200,     // ms guaranteed reaction time for gap calculation
};
```

In src/systems/difficulty.js:
1. Add module-level `let elapsedTime = 0;` alongside existing destroyCount/currentLevel.
2. Add `export function tickSpeed(dt)` that increments elapsedTime by dt. Called each frame from main.js.
3. Add `export function getCurrentSpeed()` that returns:
   `Math.min(SPEED.BASE_SPEED + SPEED.ACCELERATION * elapsedTime, SPEED.MAX_SPEED)`
4. Add `export function getMinGap(currentSpeed)`:
   Returns the minimum pixel distance between obstacles such that at currentSpeed, the player has at least MIN_REACTION_MS to react.
   Formula: `currentSpeed * (SPEED.MIN_REACTION_MS / 1000)` — then add obstacle width (GAME.OBSTACLE_WIDTH) as buffer.
   So: `currentSpeed * 0.2 + GAME.OBSTACLE_WIDTH`
   At 140 px/s: 28 + 48 = 76px. At 280 px/s: 56 + 48 = 104px.
   These are tighter than the current fixed 120px which is fine — old gap was overly generous.
   Actually keep a floor: `Math.max(GAME.OBSTACLE_WIDTH * 2.5, currentSpeed * 0.2 + GAME.OBSTACLE_WIDTH)` = at least 120px.
5. Add `export function isWarmupComplete()` that returns `elapsedTime >= SPEED.WARMUP_TIME`.
6. Modify `resetDifficulty()` to also reset `elapsedTime = 0`.
7. Modify `getDifficultyParams()` to use `getCurrentSpeed()` for scrollSpeed instead of tier-based speed. Keep tier-based spawnInterval, maxObstacles, multiplier, tallObstacles (levels still advance by destroy count).
8. Update the DEV console.table to show new speed model (time-based instead of tier-speed).
  </action>

Run `grep "SPEED" src/config.js` — should show the new section.
