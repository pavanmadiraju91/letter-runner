  <name>Task 1: HUD renderer module</name>
  <files>src/systems/hud.js</files>
  <action>
  Create src/systems/hud.js that renders the three HUD elements onto the canvas context.

  Imports:
  - `getScore` from '../systems/score.js'
  - `getLives` from '../systems/lives.js'
  - `GAME` from '../config.js'

  Module-level state:
  - `let currentLevel = 1;` — placeholder, updated by LEVEL_UP event listener
  - In module init (top-level or lazy): subscribe to 'LEVEL_UP' event to update currentLevel

  Actually, keep it simpler — export a single `renderHUD(ctx, canvasWidth)` function and a `createHUD()` init function:

  `createHUD()`:
  - Subscribe to LEVEL_UP: `currentLevel = payload.level` (future-proof for Phase 5)
  - Subscribe to GAME_RESTART: `currentLevel = 1`

  `renderHUD(ctx, canvasWidth)`:
  - All rendering uses `ctx.save()` / `ctx.restore()` to avoid polluting state
  - Padding: 16px from edges, 8px vertical from top

  **Score (top-left):**
  - `ctx.textAlign = 'left'`
  - `ctx.textBaseline = 'top'`
  - `ctx.font = 'bold 20px monospace'`
  - `ctx.fillStyle = '#ffffff'`
  - Draw: `ctx.fillText('SCORE ' + getScore(), 16, 12)`

  **Level (top-center):**
  - `ctx.textAlign = 'center'`
  - Draw: `ctx.fillText('LV ' + currentLevel, canvasWidth / 2, 12)`

  **Lives (top-right) — heart icons:**
  - `ctx.textAlign = 'right'`
  - Render hearts as filled/empty using unicode or drawn shapes
  - Use a simple approach: draw `GAME.STARTING_LIVES` heart shapes, fill red for remaining lives, fill dark grey for lost lives
  - Heart drawing: use ctx.beginPath() with a simple heart path (two arcs + triangle) at ~14px size
  - Position: start at `canvasWidth - 16` and go left, spacing hearts by 22px
  - Filled heart color: '#ff3366' (neon pink-red)
  - Empty heart color: '#2a2a3a' (dark grey, visible but subdued)
  - Draw STARTING_LIVES hearts total, first `getLives()` of them filled, rest empty

  Keep it minimal — no shadows, no animations (those come in Phase 8).
  </action>
