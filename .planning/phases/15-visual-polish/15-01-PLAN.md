<task type="auto">
  <name>Task 1: Animated player sprite with vertical bob</name>
  <files>src/entities/player.js, src/config.js</files>
  <action>
Rewrite src/entities/player.js to use the cat-run.png sprite sheet:

1. Load sprite sheet: `const spriteSheet = new Image(); spriteSheet.src = './cat-run.png';`
   - cat-run.png is 8 frames of a running cat, arranged horizontally
   - Each frame is square (sheet width / 8 = frame size)
   - Use frames 0-7 for run animation cycle

2. Animation config in src/config.js — add PLAYER section:
   ```
   export const PLAYER = {
     FRAME_COUNT: 8,
     FRAME_DURATION: 0.1,  // seconds per frame (10 FPS animation)
     BOB_AMPLITUDE: 1,      // 1px vertical bob
     BOB_SPEED: 12,         // radians/sec for bob cycle
   };
   ```

3. createPlayer() — add fields: `frameIndex: 0, frameTimer: 0, bobTimer: 0`

4. Add updatePlayer(player, dt) export:
   - Advance frameTimer by dt; when >= PLAYER.FRAME_DURATION, increment frameIndex (mod FRAME_COUNT), reset timer
   - Advance bobTimer by dt (wraps at 2*PI naturally via Math.sin)

5. renderPlayer(ctx, player) — replace pixel-by-pixel drawing with:
   ```
   if (!spriteSheet.complete) return; // graceful fallback while loading
   const frameW = spriteSheet.width / PLAYER.FRAME_COUNT;
   const frameH = spriteSheet.height;
   const srcX = player.frameIndex * frameW;
   const bobY = Math.sin(player.bobTimer * PLAYER.BOB_SPEED) * PLAYER.BOB_AMPLITUDE;
   ctx.drawImage(
     spriteSheet,
     srcX, 0, frameW, frameH,
     Math.round(player.x), Math.round(player.y + bobY),
     player.width, player.height
   );
   ```

6. Remove the old SPRITE array, SPRITE_COLORS array entirely.
  </action>
  <verify>
Open game in browser. Player should be an animated cat sprite cycling through frames, with a subtle 1px vertical bounce. No errors in console. If sprite fails to load, player area is just empty (graceful degradation).
  </verify>
  <done>Cat sprite animates through 8 frames during gameplay. 1px vertical bob visible. Old pixel array code removed.</done>
</task>

<task type="auto">
  <name>Task 2: Parallax background with tiled bg-dark.png</name>
  <files>src/entities/background.js, src/main.js</files>
  <action>
Create src/entities/background.js:

1. Load background image: `const bgImage = new Image(); bgImage.src = './bg-dark.png';`

2. Exports:
   - `createBackground()` — returns `{ offset: 0 }`
   - `updateBackground(bg, dt, scrollSpeed)` — advances offset at 10% of scrollSpeed:
     ```
     bg.offset = (bg.offset + scrollSpeed * 0.1 * dt) % (bgImage.width || 1);
     ```
   - `renderBackground(ctx, bg, canvasWidth, canvasHeight)` — tiles bg-dark.png horizontally to fill canvas:
     ```
     if (!bgImage.complete || !bgImage.width) return;
     const imgW = bgImage.width;
     const imgH = bgImage.height;
     // Scale to fill canvas height
     const scale = canvasHeight / imgH;
     const drawW = imgW * scale;
     const startX = -(bg.offset * scale) % drawW;
     for (let x = startX; x < canvasWidth; x += drawW) {
       ctx.drawImage(bgImage, x, 0, drawW, canvasHeight);
     }
     ```

3. Update src/main.js:
   - Import: `import { createBackground, updateBackground, renderBackground } from './entities/background.js';`
   - After ground creation: `const background = createBackground();`
   - In update() (playing state): `updateBackground(background, dt, params.scrollSpeed);`
   - In render(): replace `ctx.fillStyle = COLORS.BG; ctx.fillRect(0, 0, w, h);` with:
     ```
     ctx.fillStyle = COLORS.BG;
     ctx.fillRect(0, 0, w, h);  // keep as fallback base
     renderBackground(ctx, background, w, h);
     ```
   - Also add `updatePlayer(player, dt)` call in the update() function (import updatePlayer from player.js)

The background scrolls slowly (10% of game scroll speed) creating depth against the faster ground and obstacles.
