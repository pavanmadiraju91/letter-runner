<task type="auto">
  <name>Task 2: Create difficulty.js system — level tracking, tier lookup, LEVEL_UP emission</name>
  <files>src/systems/difficulty.js</files>
  <action>
Create `src/systems/difficulty.js` with the following design:

Module-level state (consistent with lives.js, score.js pattern):
- `let destroyCount = 0`
- `let currentLevel = 1`

Exports:

1. `createDifficulty()` — subscribes to events:
   - `OBSTACLE_DESTROYED`: increment destroyCount; if destroyCount reaches currentLevel * DESTROYS_PER_LEVEL, compute new level = Math.floor(destroyCount / DESTROYS_PER_LEVEL) + 1, and if newLevel > currentLevel, set currentLevel = newLevel, emit `LEVEL_UP` with `{ level: currentLevel, ...getDifficultyParams() }`.
   - `GAME_RESTART`: call resetDifficulty().

2. `resetDifficulty()` — resets destroyCount = 0, currentLevel = 1. Does NOT emit LEVEL_UP (fresh game starts at implicit level 1).

3. `getDifficultyParams()` — returns an object with current tier parameters:
   ```js
   { scrollSpeed, spawnInterval, maxObstacles, multiplier, tallObstacles }
   ```
   Logic:
   - If currentLevel <= TIERS.length: return TIERS[currentLevel - 1] directly.
   - If currentLevel > TIERS.length: use last tier as base, apply logarithmic scaling:
     - `const overflow = currentLevel - TIERS.length;`
     - `const logFactor = Math.log(overflow + 1);` // ln(1)=0, ln(2)=0.69, ln(3)=1.1...
     - `scrollSpeed = baseTier.scrollSpeed + DIFFICULTY.LOG_SPEED_FACTOR * logFactor;`
     - `spawnInterval = Math.max(DIFFICULTY.MIN_SPAWN_INTERVAL, baseTier.spawnInterval - DIFFICULTY.LOG_SPAWN_FACTOR * logFactor);`
     - `maxObstacles = Math.min(DIFFICULTY.MAX_OBSTACLES_CAP, baseTier.maxObstacles + (overflow >= 3 ? 1 : 0));` // cap at 4
     - `multiplier = baseTier.multiplier + DIFFICULTY.LOG_MULT_FACTOR * logFactor;`
     - `tallObstacles = true;`

4. `getLevel()` — returns currentLevel (for HUD or other systems that need it).

Import events from '../core/events.js' and DIFFICULTY from '../config.js'.

Do NOT import or modify score.js — this system handles its own destroy counting independently.
  </action>
